# 🏗️ Архитектура IDE и разработки плагинов
## 1. Архитектура VSCode для плагинов
VSCode использует архитектуру на основе Language Server Protocol (LSP) и Extension API:

```text
┌──────────────────────────────────────────┐
│            VSCode Core                   │
│  ┌─────────────────────────────────────┐ │
│  │         Extension Host              │ │
│  │  ┌───────────────────────────────┐  │ │
│  │  │       Sort-me Plugin          │  │ │
│  │  │  ┌─────────────────────────┐  │  │ │
│  │  │  │    CLI Commands         │  │  │ │
│  │  │  │  ┌───────────────────┐  │  │  │ │
│  │  │  │  │   API Client      │  │  │  │ │
│  │  │  │  └───────────────────┘  │  │  │ │
│  │  │  └─────────────────────────┘  │  │ │
│  │  └───────────────────────────────┘  │ │
│  └─────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```
## 2. Модель расширения VSCode
### Компоненты:

+ Extension Manifest (package.json) - метаданные плагина

+ Activation Events - события активации

+ Contribution Points - точки интеграции

+ API Surface - доступные API методы

## 3. Наш подход: CLI-based плагин
### Мы выбрали архитектуру CLI-плагина, который работает через терминал VSCode:

```go
// Архитектура нашего плагина

│                 │                      │                 │
│  ┌─────────────┐│                      └─────────────────┘
│  │ Sort-me CLI ││
│  │ Plugin      ││
│  └─────────────┘│
```

# 🔧 Реализация плагина
## 1. Технологический стек
+ Язык: Go 1.19+

+ Библиотеки:

    + cobra - CLI framework

    + viper - конфигурация

    + websocket - real-time уведомления

## 2. Структура проекта
```text
sortme_plugin/
├── main.go                 # Точка входа
├── api_client.go           # Клиент API sort-me.org
├── vscode_extension.go     # CLI команды
├── config.go              # Управление настройками
└── README.md              # Документация
```
## 3. Ключевые компоненты
### APIClient - работа с REST API и WebSocket:

```go
type APIClient struct {
    config  *Config
    client  *http.Client
    baseURL string
}
VSCodeExtension - команды CLI:

go
type VSCodeExtension struct {
    config    *Config
    apiClient *APIClient
}
```

# 📦 Установка и использование
## Установка
```bash
git clone https://github.com/your-username/sortme-plugin
cd sortme-plugin
go build -o sortme .
sudo cp sortme /usr/local/bin/
```
## Настройка
```bash
sortme auth
```
## Использование
```bash
sortme contests                    # Список контестов
sortme problems 0                 # Задачи контеста
sortme submit solution.cpp -c 0 -p 1018  # Отправка решения
sortme status 891549              # Статус отправки
🎯 Примеры работы
```
## Просмотр контестов
```bash
$ sortme contests

🏆 Найдено контестов: 9

📚 Архивные контесты:
```
